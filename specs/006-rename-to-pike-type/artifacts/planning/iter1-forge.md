# Implementation Plan — Spec 006: Rename to pike-type

## Strategy

This is a mechanical rename with no logic changes. The implementation follows a strict ordering to keep the codebase buildable at each step. Golden files are regenerated by running the renamed tool, not hand-edited.

## Steps

### Step 1: Clean build artifacts

Delete all `__pycache__/` directories and `*.egg-info/` directories:

```bash
find src/ tests/ -type d -name __pycache__ -exec rm -rf {} + 2>/dev/null
rm -rf src/*.egg-info
```

**Files affected:** None (only cache/build artifacts deleted)

### Step 2: Rename source package directory (FR-1)

```bash
git mv src/typist src/piketype
```

**Files affected:** `src/typist/` → `src/piketype/` (entire tree)

### Step 3: Update `pyproject.toml` (FR-3)

- `name = "typist"` → `name = "pike-type"`
- `typist = "typist.cli:main"` → `piketype = "piketype.cli:main"`

**Files affected:** `pyproject.toml`

### Step 4: Update all internal imports and symbols (FR-2, FR-4, FR-5, FR-7, FR-8, FR-9, FR-15)

Do a comprehensive find-and-replace across all Python files in `src/piketype/`:

1. Replace all `from typist.` → `from piketype.` and `import typist.` → `import piketype.`
2. `cli.py`: `prog="typist"` → `prog="piketype"` (FR-4)
3. `errors.py`: `TypistError` → `PikeTypeError`, update docstrings (FR-9)
4. `discovery/scanner.py`: rename functions `is_under_typist_dir` → `is_under_piketype_dir`, `find_typist_modules` → `find_piketype_modules`, update string literals `"typist"` → `"piketype"` in path checks and error messages (FR-5)
5. `paths.py`: `typist_runtime*` → `piketype_runtime*`, `typist_manifest` → `piketype_manifest` (FR-7)
6. `backends/common/headers.py`: `"Generated by typist"` → `"Generated by piketype"` (FR-6)
7. `backends/py/emitter.py`: `"Generated by typist"` → `"Generated by piketype"` (FR-6)
8. `backends/runtime/emitter.py`: all `typist_runtime` → `piketype_runtime`, `TYPIST_RUNTIME` → `PIKETYPE_RUNTIME`, `"Generated by typist"` → `"Generated by piketype"` (FR-6)
9. `backends/cpp/emitter.py`: `part != "typist"` → `part != "piketype"` (FR-6)
10. `validate/engine.py`: `"typist file defines no DSL objects"` → `"piketype file defines no DSL objects"` (FR-8)
11. `commands/gen.py`: same error message update + all caller references to renamed scanner functions (FR-8)
12. `__init__.py`: docstring `"Typist package."` → `"Pike-type package."` (FR-15)
13. All other source files: update import paths

**Files affected:** Every `.py` file in `src/piketype/`

### Step 5: Update test fixture directories (FR-10)

Rename all `typist/` directories in fixtures:

```bash
# For each fixture with alpha/typist/ or beta/typist/
find tests/fixtures -type d -name typist | while read d; do
  git mv "$d" "$(dirname "$d")/piketype"
done

# Rename outside_typist fixture
git mv tests/fixtures/outside_typist tests/fixtures/outside_piketype

# Rename not_typist directory inside it
git mv tests/fixtures/outside_piketype/project/alpha/not_typist tests/fixtures/outside_piketype/project/alpha/not_piketype
```

Then update DSL imports inside fixture Python files:
- `from typist.dsl import ...` → `from piketype.dsl import ...`

**Files affected:** All directories and `.py` files under `tests/fixtures/`

### Step 6: Update golden file directories and names (FR-11)

Rename all golden file directories and files containing `typist`:

```bash
# Rename typist_manifest.json files
find tests/goldens -name 'typist_manifest.json' | while read f; do
  git mv "$f" "$(dirname "$f")/piketype_manifest.json"
done

# Rename typist/ directories in goldens (py/alpha/typist, sv/alpha/typist, etc.)
find tests/goldens -type d -name typist | while read d; do
  git mv "$d" "$(dirname "$d")/piketype"
done

# Rename typist_runtime* files
find tests/goldens -name 'typist_runtime*' | while read f; do
  newname=$(echo "$f" | sed 's/typist_runtime/piketype_runtime/')
  git mv "$f" "$newname"
done
```

**Files affected:** All files under `tests/goldens/gen/`

### Step 7: Regenerate golden file content

After Steps 4-6, install the package in dev mode and regenerate goldens by running `piketype gen` on each fixture:

```bash
uv pip install -e .
# For each fixture, run piketype gen and copy output to goldens
```

This ensures golden file content (headers, manifest JSON, import paths) matches the new names.

**Files affected:** Content of all golden files under `tests/goldens/gen/`

### Step 8: Update test source files (FR-12)

Update all `tests/test_*.py` files:
- Imports: `from typist.*` → `from piketype.*`
- CLI invocations: `-m typist.cli` → `-m piketype.cli`
- Fixture paths: `"typist"` → `"piketype"`, `"outside_typist"` → `"outside_piketype"`, `"not_typist"` → `"not_piketype"`
- Method names: `run_typist` → `run_piketype`, etc.
- String literals in test data: module names, namespace parts, paths

**Files affected:** All `tests/test_*.py` files

### Step 9: Update documentation (FR-13)

Replace all `typist` references in:
- `README.md`
- `docs/rfc-v1.md`
- `docs/architecture.md`
- `docs/v1-product-spec.md`
- `docs/milestone-01-const-sv.md`
- `docs/requirements.org`
- `docs/planning.org`
- `docs/ir-schema.md`

All `typist_` prefixes become `piketype_`. CLI command references become `piketype`. Project name references become `pike-type`.

**Files affected:** `README.md`, all files in `docs/`

### Step 10: Update project constitution (FR-14)

Update `.steel/constitution.md`:
- `src/typist/` → `src/piketype/`
- `typist gen` → `piketype gen`
- `typist/` modules → `piketype/` modules
- All other references

**Files affected:** `.steel/constitution.md`

### Step 11: Regenerate `uv.lock` (FR-16)

```bash
uv lock
```

**Files affected:** `uv.lock`

### Step 12: Verify (AC-1 through AC-14)

Run all acceptance criteria checks:

```bash
# AC-1: Install and verify CLI
uv pip install -e .
which piketype

# AC-3: Run tests
python3 -m unittest discover -s tests
python3 -m pytest

# AC-4: No stale references
grep -rI -i "typist" --exclude-dir=__pycache__ --exclude-dir='*.egg-info' --exclude='*.pyc' src/ tests/ pyproject.toml README.md docs/ .steel/constitution.md

# AC-9: Type checking
basedpyright src/piketype/

# AC-11: Import check
python3 -c "import piketype; print(piketype.__version__)"

# AC-13: uv.lock check
grep 'name = "pike-type"' uv.lock

# AC-14: No stale paths
find src/ tests/ -name '*typist*'
```

## Risk Assessment

| Risk | Mitigation |
|------|------------|

| Missed stale path | AC-14 find catches any misses |

| Broken imports | basedpyright strict mode (AC-9) |
| Test failures | Full test suite run (AC-3) |
