# Implementation Plan — Spec 006: Rename to pike-type

## Strategy

This is a mechanical rename with no logic changes. Steps 1-3 form an atomic block — they must all complete before any verification is possible. Golden files are regenerated by running the renamed tool on positive fixtures only.

## Steps

### Step 1: Clean build artifacts (NFR-5)

```bash
find src/ tests/ -type d -name __pycache__ -exec rm -rf {} + 2>/dev/null
rm -rf src/*.egg-info
```

**Files affected:** None tracked (only cache/build artifacts deleted)

### Step 2: Atomic package rename (FR-1, FR-2, FR-3, FR-4, FR-5, FR-6, FR-7, FR-8, FR-9, FR-15)

This step is atomic — all changes happen together before any test or build is attempted.

**2a. Rename source directory:**
```bash
git mv src/typist src/piketype
```

**2b. Update `pyproject.toml` (FR-3):**
- `name = "typist"` → `name = "pike-type"`
- `typist = "typist.cli:main"` → `piketype = "piketype.cli:main"`

**2c. Update all source code in `src/piketype/`:**

Across every `.py` file in `src/piketype/`:
1. Replace all `from typist.` → `from piketype.` and `import typist.` → `import piketype.` (FR-2)
2. `cli.py`: `prog="typist"` → `prog="piketype"` (FR-4)
3. `errors.py`: `TypistError` → `PikeTypeError`, docstring `"Base exception for typist."` → `"Base exception for piketype."` (FR-9)
4. `discovery/scanner.py`: rename `is_under_typist_dir` → `is_under_piketype_dir`, `find_typist_modules` → `find_piketype_modules`, `"typist"` → `"piketype"` in path checks and error messages (FR-5)
5. `paths.py`: `typist_runtime*` → `piketype_runtime*`, `typist_manifest` → `piketype_manifest` (FR-7)
6. `backends/common/headers.py`: `"Generated by typist"` → `"Generated by piketype"` (FR-6)
7. `backends/py/emitter.py`: `"Generated by typist"` → `"Generated by piketype"` (FR-6)
8. `backends/runtime/emitter.py`: all `typist_runtime` → `piketype_runtime`, `TYPIST_RUNTIME` → `PIKETYPE_RUNTIME`, `"Generated by typist"` → `"Generated by piketype"` (FR-6)
9. `backends/cpp/emitter.py`: `part != "typist"` → `part != "piketype"` (FR-6)
10. `validate/engine.py`: `"typist file defines no DSL objects"` → `"piketype file defines no DSL objects"` (FR-8)
11. `commands/gen.py`: same error message update + `find_typist_modules` → `find_piketype_modules` (FR-8)
12. `__init__.py`: docstring `"Typist package."` → `"Pike-type package."` (FR-15)

**Files affected:** `pyproject.toml`, every `.py` file in `src/piketype/`

### Step 3: Update test fixtures (FR-10)

**3a. Rename directories:**
```bash
# Rename all typist/ directories in fixtures to piketype/
find tests/fixtures -type d -name typist | while read d; do
  git mv "$d" "$(dirname "$d")/piketype"
done

# Rename negative fixture directory
git mv tests/fixtures/outside_typist tests/fixtures/outside_piketype

# Rename not_typist directory inside it
git mv tests/fixtures/outside_piketype/project/alpha/not_typist tests/fixtures/outside_piketype/project/alpha/not_piketype
```

**3b. Update DSL imports in fixture Python files:**
- `from typist.dsl import ...` → `from piketype.dsl import ...` (all fixture `.py` files)

**Files affected:** All directories and `.py` files under `tests/fixtures/`

### Step 4: Rename golden file paths (FR-11)

Rename all golden file directories and files containing `typist`:

```bash
# Rename typist_manifest.json files
find tests/goldens -name 'typist_manifest.json' | while read f; do
  git mv "$f" "$(dirname "$f")/piketype_manifest.json"
done

# Rename typist/ directories in goldens
find tests/goldens -type d -name typist | while read d; do
  git mv "$d" "$(dirname "$d")/piketype"
done

# Rename typist_runtime* files
find tests/goldens -name 'typist_runtime*' | while read f; do
  newname=$(echo "$f" | sed 's/typist_runtime/piketype_runtime/')
  git mv "$f" "$newname"
done
```

**Files affected:** All paths under `tests/goldens/gen/`

### Step 5: Regenerate golden file content (FR-11)

Install the package and regenerate goldens for all **positive** fixtures only. Negative fixtures (`no_dsl`, `outside_piketype`, `const_collision`, `struct_pad_collision`, `scalar_signed_wide`) do not have golden output.

The 14 positive fixtures with goldens:
1. `const_cpp_explicit_uint32`
2. `const_cpp_wide`
3. `const_expr_basic`
4. `const_sv_basic`
5. `flags_basic`
6. `namespace_override`
7. `nested_struct_sv_basic`
8. `scalar_sv_basic`
9. `scalar_wide`
10. `struct_multiple_of`
11. `struct_padded`
12. `struct_signed`
13. `struct_sv_basic`
14. `struct_wide`

For each, copy the fixture to a temp dir, run `piketype gen` (with `--namespace` for `namespace_override`), and copy the `gen/` output to `tests/goldens/gen/<case>/`.

**Files affected:** Content of all golden files under `tests/goldens/gen/`

### Step 6: Update test source files (FR-12)

Update all `tests/test_*.py` files:
- Imports: `from typist.*` → `from piketype.*`
- CLI invocations: `-m typist.cli` → `-m piketype.cli`
- Fixture paths: `"typist"` → `"piketype"`, `"outside_typist"` → `"outside_piketype"`, `"not_typist"` → `"not_piketype"`
- Method names: `run_typist` → `run_piketype`, etc.
- String literals in test data: module names, namespace parts, paths
- Expected error messages referencing "typist" → "piketype"

**Files affected:** All `tests/test_*.py` files

### Step 7: Update documentation (FR-13)

Replace all `typist` references in all doc files. All `typist_` prefixes become `piketype_`. CLI command references become `piketype`. Project name references become `pike-type`.

Files to update:

- `docs/rfc-v1.md`
- `docs/architecture.md`
- `docs/v1-product-spec.md`
- `docs/milestone-01-const-sv.md`
- `docs/requirements.org`

- `docs/ir-schema.md`

**Files affected:** `README.md`, all files in `docs/`

### Step 8: Update project constitution (FR-14)

Update `.steel/constitution.md`:
- `src/typist/` → `src/piketype/`
- `typist gen` → `piketype gen`
- `typist/` modules → `piketype/` modules

**Files affected:** `.steel/constitution.md`

### Step 9: Regenerate `uv.lock` (FR-16)

```bash

```

**Files affected:** `uv.lock`

### Step 10: Verify all acceptance criteria

```bash
# NFR-5: Clean artifacts
find src/ tests/ -type d -name __pycache__ -exec rm -rf {} + 2>/dev/null
rm -rf src/*.egg-info

# AC-1: Install and verify CLI
pip install -e .
which piketype

# AC-3: Run tests (primary: unittest per constitution, secondary: pytest)
python3 -m unittest discover -s tests
python3 -m pytest

# AC-4: No stale references in file contents
! grep -rI -i "typist" --exclude-dir=__pycache__ --exclude-dir='*.egg-info' --exclude='*.pyc' src/ tests/ pyproject.toml README.md docs/ .steel/constitution.md

# AC-9: Type checking
basedpyright src/piketype/

# AC-11: Import check
python3 -c "import piketype; print(piketype.__version__)"

# AC-12: Exception class hierarchy
python3 -c "
from piketype.errors import PikeTypeError, DiscoveryError, ValidationError, EmissionError
assert issubclass(DiscoveryError, PikeTypeError)
assert issubclass(ValidationError, PikeTypeError)
assert issubclass(EmissionError, PikeTypeError)
print('AC-12: PASS')
"

# AC-13: uv.lock verification
grep -q 'name = "pike-type"' uv.lock && ! grep -q 'name = "typist"' uv.lock && echo "AC-13: PASS"

# AC-14: No stale paths
test -z "$(find src/ tests/ -name '*typist*' 2>/dev/null)" && echo "AC-14: PASS"
```

## Risk Assessment

| Risk | Mitigation |
|------|------------|
| Missed stale reference | AC-4 grep catches any misses |
| Missed stale path | AC-14 find catches any misses |
| Golden file content mismatch | Regeneration in Step 5 using only positive fixtures |
| Broken imports | basedpyright strict mode (AC-9) |
| Test failures | Full test suite (AC-3) |
| Intermediate inconsistency | Steps 2a-2c are atomic — no test/build between them |
