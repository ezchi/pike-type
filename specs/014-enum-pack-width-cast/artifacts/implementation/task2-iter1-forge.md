## Key Implementation Decisions
- **Used the project-standard regen pattern** (per
  `project_golden_regen_pattern.md`): copy fixture project → temp dir,
  run `python -m piketype.cli gen <cli_file>`, then sync back. The
  invocation matches `tests/test_gen_*.py:_run_piketype` /
  `_gen_fixture` helpers.
- **Used the venv's Python interpreter** (`/Users/ezchi/Projects/pike-type/.venv/bin/python`), not system `python3`, because Jinja2 is
  not installed in the system Python. The system tests run via this
  same venv.
- **For `cross_module_type_refs_namespace_proj`, used the
  `cross_module_type_refs` fixture with the extra arg
  `--namespace=proj::lib`** (per the per-fixture invocation table in
  `tasks.md` and `tests/test_gen_cross_module.py:95-107`). There is no
  `cross_module_type_refs_namespace_proj` fixture directory; the
  golden is generated by re-invoking the same fixture with the
  namespace flag.
- **Synced only the changed files individually**, not whole subtrees,
  to keep the diff minimal and to avoid accidentally pulling in
  unintended side effects (e.g. stray `__pycache__/` directories that
  exist under some goldens).
- **Verified pre-replace diff scope before syncing**: ran
  `diff -r tests/goldens/gen/<fixture> /tmp/pike-014/<fixture>/gen`
  for each fixture and confirmed only `.sv` lines `13` (or `46`, for
  `foo_pkg.sv`) differ — no other lines, no other files.

## Deviations from Plan
- **The plan's Phase 2 illustrative shell loop assumed the fixture
  directory and golden directory share a name.** This is true for 4 of
  5 fixtures, but `cross_module_type_refs_namespace_proj` has no
  fixture of that name; it reuses the `cross_module_type_refs`
  fixture with `--namespace=proj::lib`. The Tasks document's "Plan
  corrections required" section flagged this; the implementation
  followed the corrected per-fixture invocation table.
- **Synced individual files instead of whole `sv/` subtrees.** The
  plan's Phase 2 said "overwrite the corresponding subtree under
  `tests/goldens/gen/`" — the implementation chose file-level sync to
  reduce risk of pulling unintended differences in other generated
  files (cpp, py, manifest). The end result is identical: the same
  files end up byte-equal to the freshly generated output. The
  finer-grained sync is safer.
